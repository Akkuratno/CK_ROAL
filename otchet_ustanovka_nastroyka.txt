================================================================================
1. ВВЕДЕНИЕ
================================================================================

Отчёт описывает порядок установки и настройки разработанного веб-приложения «Сайт строительной компании Роял» на локальный или серверный компьютер. Цель — обеспечить корректный запуск сайта, работу формы обратной связи и сохранение заявок.

Результат установки: пользователь может открыть сайт в браузере по заданному адресу, просматривать разделы (О нас, Услуги, Проекты, Преимущества, Контакты) и отправлять заявки через форму; заявки сохраняются в файл и при необходимости отправляются на электронную почту.

================================================================================
2. ТРЕБОВАНИЯ К ПРОГРАММНОМУ ОБЕСПЕЧЕНИЮ
================================================================================

Для работы веб-приложения необходимо следующее ПО:

| Компонент        | Требование |
|------------------|------------|
| Операционная система | Windows 10/11, Linux или серверная ОС с веб-сервером |
| Веб-сервер       | Apache 2.4 или аналог (входит в XAMPP, OpenServer, Denwer и т.п.) |
| Интерпретатор PHP | PHP 7.4 или выше (рекомендуется PHP 8.x) |
| Браузер          | Любой современный браузер (Chrome, Firefox, Edge, Safari) с поддержкой HTML5, CSS3, JavaScript |

Примечание: база данных не используется. Заявки сохраняются в файл leads.json. Отправка писем на почту выполняется через стандартную функцию PHP mail() и зависит от настройки почты на сервере.

================================================================================
3. СОСТАВ ПОСТАВКИ
================================================================================

В состав веб-приложения входят следующие файлы:

| Файл         | Назначение |
|--------------|------------|
| index.php    | Главная страница сайта (разметка всех разделов) |
| contact.php  | Обрабатывает заявку с формы: принимает данные, проверяет поля, сохраняет в файл leads.json, при желании отправляет письмо на почту |
| style.css    | Таблица стилей: тема, типографика, сетки, адаптивность, стили формы и компонентов |
| script.js    | Скрипты: мобильное меню (бургер), плавная прокрутка по якорям, отправка формы через fetch, анимация счётчиков |

Внешние ресурсы (подключаются по сети):
- Шрифт Inter — Google Fonts (https://fonts.googleapis.com).
- Иконки Font Awesome — CDN (https://cdnjs.cloudflare.com).

Файл leads.json создаётся автоматически при первой отправке заявки; при отсутствии прав на запись в каталог приложения его можно создать вручную (пустой массив: []).

================================================================================
4. УСТАНОВКА
================================================================================

4.1. Размещение файлов

1. Скопировать в каталог веб-сервера все файлы приложения (index.php, contact.php, style.css, script.js).
2. При использовании XAMPP — поместить их в подкаталог папки htdocs, например:
   C:\xampp\htdocs\roal\
   В результате главная страница будет доступна по адресу: http://localhost/roal/
3. При размещении на хостинге — загрузить файлы в корневой каталог сайта или в нужную подпапку (например, public_html или www) в соответствии с инструкциями хостинг-провайдера.

4.2. Права доступа (для Linux-серверов)

- Каталог с сайтом должен быть доступен для чтения веб-серверу (пользователь www-data, apache или nginx).
- Для записи заявок в leads.json каталог приложения должен допускать создание и изменение файлов процессом веб-сервера. Рекомендуется: chmod 755 на каталог; файл leads.json после создания — 644, владелец — пользователь веб-сервера.

4.3. Проверка после копирования

- Убедиться, что в каталоге присутствуют index.php, contact.php, style.css, script.js.
- Убедиться, что веб-сервер и PHP запущены (для XAMPP — Apache в панели управления XAMPP).

================================================================================
5. НАСТРОЙКА
================================================================================

5.1. Общие настройки

Сайт не использует отдельный конфигурационный файл. Все настройки заданы в коде.

5.2. Форма обратной связи (contact.php)

- Сохранение заявок: заявки записываются в файл leads.json в той же папке, где лежит contact.php. Ничего настраивать не нужно, если в эту папку разрешена запись.

- Отправка на почту: в contact.php прописан адрес, куда уходят письма: hertvvv@mail.ru. Чтобы изменить адрес — открыть contact.php и поменять значение переменной $to на нужный email. Письма отправляются через встроенную в PHP отправку почты; для этого на сервере должна быть настроена почта. В XAMPP по умолчанию письма могут не уходить без настройки — заявки при этом по-прежнему сохраняются в leads.json.

5.3. Контактные данные на странице

Телефон, адрес, email и режим работы отображаются в разметке index.php (блок контактов и подвал). Для изменения — отредактировать соответствующие фрагменты в index.php.

================================================================================
6. ПРОВЕРКА РАБОТОСПОСОБНОСТИ
================================================================================

6.1. Открытие главной страницы

- В браузере открыть адрес сайта (например, http://localhost/roal/).
- Должна отображаться главная страница: шапка с логотипом «СК РОЯЛ», меню, главный экран с текстом «Строим будущее вместе с вами», далее разделы О нас, Услуги, Проекты, Преимущества, Контакты.

6.2. Навигация и адаптивность

- Проверить переход по пунктам меню (О нас, Услуги, Проекты, Преимущества, Контакты): страница должна плавно прокручиваться к соответствующему разделу.
- Уменьшить ширину окна браузера или включить режим эмуляции мобильного устройства (F12 → переключение устройства): должна появляться кнопка-бургер, по нажатию — открываться меню с теми же пунктами.

6.3. Форма обратной связи

- Перейти к разделу «Контакты» (якорь #contacts).
- Заполнить обязательные поля «Имя» и «Телефон», при желании — «Email» и «Сообщение».
- Нажать «Отправить заявку».
- Должно появиться сообщение об успешной отправке (или об ошибке при сбое).
- Проверить наличие или создание файла leads.json в каталоге сайта и наличие в нём новой записи с введёнными данными.

6.4. Краткий чек-лист

| Проверка                          | Ожидаемый результат |
|-----------------------------------|---------------------|
| Открытие главной страницы         | Страница загружается, стили и шрифты отображаются |
| Плавная прокрутка по меню         | Клик по пункту меню прокручивает к разделу |
| Бургер-меню на узком экране       | Меню открывается и закрывается по клику |
| Отправка формы (имя + телефон)    | Сообщение «Заявка успешно отправлена», запись в leads.json |

================================================================================
7. ВОЗМОЖНЫЕ ПРОБЛЕМЫ И РЕШЕНИЯ
================================================================================

| Проблема | Возможная причина | Решение |
|----------|-------------------|---------|
| Страница не открывается | Сервер не запущен | Включить Apache (в XAMPP нажать Apache → Start) |
| Ошибка при отправке формы | Папке или файлу нельзя записывать | Разрешить запись в папку сайта или создать пустой файл leads.json и разрешить в него запись |
| Письма не приходят на почту | На сервере не настроена отправка почты | Настроить отправку почты на сервере или пользоваться только сохранением заявок в leads.json |
| Шрифты и иконки не загружаются | Нет интернета или сайты с ресурсами заблокированы | Проверить интернет и возможность открыть в браузере fonts.googleapis.com и cdnjs.cloudflare.com |

================================================================================
8. ЗАКЛЮЧЕНИЕ
================================================================================

Чтобы установить и настроить сайт «Роял», нужно положить файлы в папку веб-сервера и при необходимости поменять в contact.php адрес почты, куда приходят заявки, и разрешить запись в папку сайта. Нужны веб-сервер с PHP (например, Apache из XAMPP) и PHP не ниже 7.4. База данных не нужна — заявки пишутся в файл leads.json.

После выполнения описанных шагов сайт готов к использованию: просмотр разделов, навигация и отправка заявок через форму обратной связи должны работать в соответствии с техническим заданием.
